Documentacion de la Unidad 1. Este proyecto integra Python, Pruebas y Sandbox.

1. Elementos de python

Esta sección contiene la documentación inicial del archivo lavadero.py, realizada al comienzo del desarrollo. En ella se describe la estructura básica de la clase Lavadero y sus componentes principales.

Este documento representa la primera aproximación a la comprensión del sistema, sentando las bases para el posterior trabajo de implementación, depuración y pruebas que se desarrollará en las siguientes etapas del proyecto.

2. Ejecucion y depuracion del codigo

Esta documentación describe el proceso de ejecución y depuración del código principal del sistema de lavadero automatizado. El objetivo fue validar el funcionamiento integral de la aplicación mediante la ejecución controlada de los casos de uso definidos.

Inicialmente, se preparó el entorno de desarrollo creando y activando un entorno virtual para aislar las dependencias del proyecto. Posteriormente, se ejecutó el programa principal identificando un error en uno de los ejemplos de uso. Tras analizar el fallo, se corrigió un parámetro incorrecto en la llamada a la función, lo que permitió que todos los ejemplos se ejecutaran correctamente y mostraran los resultados esperados.

3. Pruebas

Esta documentación describe el proceso de implementación y validación de las pruebas unitarias para el sistema de control de un lavadero automatizado. El objetivo principal fue ejecutar y depurar un conjunto de 14 tests predefinidos que verifican el comportamiento completo del sistema, desde su estado inicial hasta los flujos de trabajo con diversas opciones de servicio.

La ejecución exitosa de todos los tests al final del proceso confirma que el sistema cumple con los requisitos funcionales, maneja correctamente las excepciones y calcula con precisión los ingresos según las opciones de servicio seleccionadas por el cliente.

4. Sandbox

Esta documentación describe el proceso de ejecución de código Python en un entorno sandbox utilizando Firejail. El objetivo fue crear un ambiente aislado y seguro para probar aplicaciones sin afectar al sistema principal.

El uso de Firejail permitió ejecutar código de manera segura, manteniendo el sistema host protegido de posibles efectos no deseados durante las pruebas de desarrollo.

5. Conclusiones

Esta reflexión analiza la relación entre los lenguajes de programación y la seguridad del software, tomando como punto de partida la metáfora de Bruce Schneier sobre las cerraduras y los lenguajes. Tras estudiar el material de la Unidad 1, se examina cómo diferentes lenguajes implementan o no las medidas de protección integradas y cómo estas afectan la seguridad final de las aplicaciones.

Finalmente, se presenta una tabla comparativa que resume las características de seguridad, vulnerabilidades comunes y recomendaciones de uso para cada lenguaje analizado.
